# Функционал ограничения количества ходов и определения победителя по очкам

## Описание

Добавлен функционал для мотивирования нейросети активнее играть вместо просто передвижения фишек. Игра может закончиться двумя способами:

1. **Традиционная победа** — один из игроков набирает 6 очков
   - Value = ±1.0 (традиционное значение Alpha Zero)

2. **Победа по очкам при лимите ходов** — 200 ходов пройдено без изменения счета
   - Value = ±0.8 (меньше, чем традиционная победа)
   - Победитель определяется по количеству набранных очков
   - Ничья (Value = 0) если очки одинаковые

## Сумма ходов

Лимит составляет **200 ходов БЕЗ ИЗМЕНЕНИЯ СЧЕТА**. Это означает:
- Если счет менялся в ходе 50, а потом последующие 200 ходов счет не менялся → игра завершится после хода 250
- Если счет менялся в ходе 180, а потом 20 ходов счет не менялся → игра не завершится (нужно 200 ходов БЕЗ изменения)
- Если счет вообще не менялся → игра завершится после хода 200

## Реализация

### GameState (`src/core/game_state.py`)

Добавлены поля:
- `move_count: int` — счетчик всех сделанных ходов
- `last_score_change_move: int` — номер хода, когда последний раз изменился счет

Добавлены константы:
- `MAX_MOVES = 200` — лимит ходов
- `MOVES_LIMIT_VALUE = 0.8` — value при окончании по лимиту ходов

Новые методы:
- `is_moves_limit_reached()` — проверяет лимит ходов
- `is_game_ended()` — проверяет окончание игры (победой или по лимиту)
- `get_game_end_type()` — возвращает тип окончания ('score_win', 'moves_limit', 'not_ended')
- `get_value_for_player(player_team)` — возвращает value для игрока

### AbalonAiGameState (`src/aiBot/AbalonAiGameState.py`)

Обновлены методы:
- `getGameEnded(board, player)` — теперь возвращает value (float | None) вместо boolean
  - `None`: игра не закончилась
  - `1.0`: текущий игрок выиграл традиционным способом (6 очков)
  - `-1.0`: текущий игрок проиграл традиционным способом
  - `0.8`: текущий игрок выиграл по очкам при лимите ходов
  - `-0.8`: текущий игрок проиграл по очкам при лимите ходов
  - `0.0`: ничья при лимите ходов (одинаковый счет)

- `getNextState(board, player, action)` — увеличивает счетчик ходов и обновляет `last_score_change_move`

## Использование в Coach.py

Существующий код в [Coach.py](src/aiBot/base_alpha_zero/Coach.py#L72) обновлен для работы с `None`:

```python
r = self.game.getGameEnded(board, self.curPlayer)
if r is not None:  # Вместо: if r != 0:
    return [(x[0], x[2], r * ((-1) ** (x[1] != self.curPlayer))) for x in trainExamples]
```

Теперь корректно обрабатывает все случаи:
- `None` → игра продолжается, цикл не прерывается
- Любое другое значение (включая `0.0` при ничье) → игра закончилась, результаты добавляются в обучающий набор

## Примеры значений Value

| Сценарий | Value (Black) | Value (White) |
|----------|---------------|---------------|
| Игра не закончилась | `None` | `None` |
| Black набрал 6 очков | 1.0 | -1.0 |
| White набрал 6 очков | -1.0 | 1.0 |
| 200 ходов, Black выигрывает по очкам | 0.8 | -0.8 |
| 200 ходов, White выигрывает по очкам | -0.8 | 0.8 |
| 200 ходов, ничья (одинаковый счет) | 0.0 | 0.0 |

## Интеграция с API

Если используется [api.py](src/api/api.py) с методом `GameState.move()`, счетчик ходов также увеличивается:
- После каждого успешного хода `move_count` увеличивается
- `last_score_change_move` обновляется при изменении счета

## Тестирование

Рекомендуется протестировать:
1. Окончание игры по традиционной победе (value = ±1.0)
2. Окончание игры по лимиту ходов с разными счетами (value = ±0.8)
3. Ничья при лимите ходов (value = 0.0)
4. Незавершенная игра (value = `None`)
5. Корректность различия между `None` (игра продолжается) и `0.0` (ничья)
6. Корректность `move_count` и `last_score_change_move`

**Важно:** В Coach.py используется проверка `if r is not None:`, которая корректно отличает:
- Игру в процессе (`r = None`) → продолжение цикла
- Ничью (`r = 0.0`) → завершение эпизода и добавление результатов
