# Сбор и анализ метрик обучения

Система автоматически собирает метрики во время обучения и логирует их в консоль и JSON файл.

## Какие метрики собираются?

### Метрики каждой игры (партии)
- **Количество ходов** — сколько ходов потребовалось до завершения
- **Счет игроков** — финальный счет белых и черных
- **Способ окончания** — традиционная победа (6 очков) или по лимиту (200 ходов)
- **Победитель** — кто выиграл (White, Black, Draw)
- **Value** — получаемое вознаграждение (±1.0, ±0.8, или 0.0)

### Метрики итерации (агрегированные)
- **Средняя длина игры** — среднее количество ходов
- **Min/Max ходов** — минимум и максимум ходов в партиях
- **Результаты** — количество побед белых/черных и ничьих
- **Проценты** — % побед белых/черных от всех игр
- **Типы побед** — сколько побед по очкам и сколько по лимиту
- **Средний счет** — среднее количество очков за партию для каждого игрока
- **Loss нейросети** — ошибка обучения
- **Арена** — результаты сравнения новой модели со старой
- **Статус модели** — принята ли новая модель

## Где смотреть метрики?

### 1. Консоль (реал-тайм)
После каждой партии выводятся логи типа:
```
[Ход 1] Ход: Белые | Счет: Белые=0 Черные=0 | Ходов всего: 1/200
[Ход 2] Ход: Черные | Счет: Белые=0 Черные=0 | Ходов всего: 2/200
...
🏁 Эпизод закончен ТРАДИЦИОННОЙ ПОБЕДОЙ! Победитель: Белые
```

После каждой итерации выводится сводка:
```
================================================================================
📊 МЕТРИКИ ИТЕРАЦИИ #1
================================================================================
🎮 ИГРЫ (5 эпизодов):
   • Ходов: 125.3 ± [89, 195]
   • Результаты: ⚪ 3 | ⚫ 2 | 🤝 0
   • Проценты: ⚪ 60.0% | ⚫ 40.0%
   • Способ выигрыша: По очкам (6)=3, По лимиту (200)=2
📈 СЧЕТ:
   • Белые: 3.40 ± очков/партию
   • Черные: 2.60 ± очков/партию
🧠 ОБУЧЕНИЕ НЕЙРОСЕТИ:
   • Loss: 0.2543
⚔️  АРЕНА (сравнение моделей):
   • Новая модель: 6 побед
   • Старая модель: 2 побед
   • Ничьих: 2
   • ✅ МОДЕЛЬ ПРИНЯТА

```

## Пример анализа метрик

### Рост среднего количества ходов со временем
Если в начале игры короткие (~80 ходов), а потом становятся длинные (~180 ходов) — это плохо, значит бот не находит выигрыш.

### Процент побед белых vs черных
Если один из игроков выигрывает намного чаще — возможна проблема с несимметричностью позиции или ошибка в коде.

### Способ выигрыша
- Много побед по лимиту → игроки играют осторожно или не находят выигрыш
- Много побед по очкам → игра развивается нормально

### Loss нейросети
- Убывает → нейросеть учится хорошо
- Вырастает → возможно переобучение или проблема с данными
- Не меняется → нейросеть не учится

### Результаты арены
- Новая модель выигрывает > 55% → прогресс в обучении
- Новая модель выигрывает < 55% → старая модель лучше, нужно менять параметры

## Как использовать метрики в Jupyter?

```python
import json
import pandas as pd

# Загрузить метрики
with open('./metrics/metrics.json') as f:
    data = json.load(f)

# Извлечь данные по итерациям
iterations_df = pd.DataFrame([
    {
        'Iteration': it['iteration'],
        'Avg Moves': it['avg_moves_per_game'],
        'White Wins': it['wins_white'],
        'Black Wins': it['wins_black'],
        'Draws': it['draws'],
        'Loss': it['train_loss'],
    }
    for it in data['iterations']
])

# Построить графики
iterations_df.plot(x='Iteration', y=['Avg Moves', 'Loss'], subplots=True)
```

## Советы по анализу

1. **Ищите тренды, не точки** — одна плохая итерация — нормально, проблема если тренд устойчивый
2. **Коррелируйте метрики** — если loss падает, но % побед не растет — что-то не так
3. **Сравнивайте с базовой линией** — сохраняйте первые метрики как baseline
4. **Анализируйте отдельные игры** — если видите странный результат, посмотрите конкретную игру в `games` массиве
