# 📊 Система сбора метрик обучения

## 🚀 Быстрый старт

Метрики собираются **автоматически** во время обучения. Просто запустите:

```bash
uv run ./src/aiBot/train_bot.py
```

И вы будете видеть логи в консоли + данные будут сохраняться в `./metrics/metrics_YYYY-MM-DD_HH-MM-SS.json`

### 📁 Где хранятся данные?

При каждом запуске создаются **новые файлы**:

- **Логи:** `./logs/app_YYYY-MM-DD_HH-MM-SS.log` (все операции)
- **Метрики:** `./metrics/metrics_YYYY-MM-DD_HH-MM-SS.json` (полные данные)

Это позволяет сравнивать разные запуски и не потерять старые данные!

## 📈 Что собирается?

### Per-game метрики
- ✅ Количество ходов
- ✅ Финальный счет (белые vs черные)
- ✅ Способ окончания (по очкам или по лимиту)
- ✅ Победитель (White/Black/Draw)
- ✅ Value (награда для обучения: ±1.0, ±0.8, 0.0)

### Per-iteration метрики (агрегированные)
- ✅ **Ходы:** среднее, минимум, максимум за итерацию
- ✅ **Результаты:** кол-во побед белых/черных, ничьей, %
- ✅ **Способы побед:** по очкам (6) vs по лимиту (200)
- ✅ **Средний счет:** очки белых/черных за игру
- ✅ **Loss:** ошибка нейросети за итерацию
- ✅ **Арена:** победы новой vs старой модели, статус принятия

## 📊 Интерпретация результатов

| Метрика | Хорошо | Плохо | Что делать |
|---------|--------|-------|-----------|
| **Avg Moves** | 100-150 | > 180 | Бот не находит выигрыш |
| **Loss** | Падает | Растет | Менять LR или regularization |
| **White % wins** | ~50% | < 30% | Несимметричность, проверить код |
| **New model wins** | > 55% | < 55% | Прогресс есть / нет прогресса |
| **Draws** | < 10% | > 50% | Игра зависает в ничьях |

## 📁 Структура выходных файлов

```
logs/
├── app_2026-04-28_10-30-45.log    # Первый запуск
├── app_2026-04-28_14-15-22.log    # Второй запуск
└── app_2026-04-28_16-45-18.log    # Третий запуск

metrics/
├── metrics_2026-04-28_10-30-45.json    # Первый запуск (200 итераций)
├── metrics_2026-04-28_14-15-22.json    # Второй запуск (150 итераций)
└── metrics_2026-04-28_16-45-18.json    # Третий запуск (текущий)
```

**Преимущества:**
- ✅ Каждый запуск изолирован
- ✅ Легко сравнивать разные конфигурации
- ✅ История всегда сохранена
- ✅ Нет перезаписи старых данных

## 🐍 Использование в Python/Jupyter

```python
import json
import pandas as pd
import matplotlib.pyplot as plt
from pathlib import Path

# Найти самый свежий файл метрик
metrics_dir = Path('./metrics')
metrics_files = sorted(metrics_dir.glob("metrics_*.json"), key=lambda p: p.stat().st_mtime, reverse=True)
latest_file = metrics_files[0]

# Загрузить
with open(latest_file) as f:
    data = json.load(f)

print(f"Анализирую: {latest_file.name}")

# В DataFrame
df = pd.DataFrame([
    {
        'iter': it['iteration'],
        'avg_moves': it['avg_moves_per_game'],
        'w_wins': it['wins_white'],
        'loss': it['train_loss'],
        'accepted': it['model_accepted'],
    }
    for it in data['iterations']
])

# Графики
df.set_index('iter')[['avg_moves', 'loss']].plot(subplots=True)
plt.show()
```

## 💡 Советы по анализу

### 🔴 Проблемы и решения

**Проблема:** Loss не падает, модели отклоняются
```
Решение:
1. Увеличить numEps (больше примеров)
2. Уменьшить learning rate
3. Проверить, что генируются ли разные позиции
```

**Проблема:** Ходов становится всё больше (> 180)
```
Решение:
1. Бот слишком осторожный
2. Проверить функцию оценки стратегии
3. Может быть проблема в generate_moves
```

**Проблема:** Всё время ничьи (> 50%)
```
Решение:
1. Увеличить MOVES_LIMIT_VALUE (мотивация на активную игру)
2. Уменьшить MAX_MOVES (короче игра)
3. Проверить balance между игроками
```

### ✅ Признаки хорошего обучения

- Loss медленно падает по итерациям
- Новые модели выигрывают > 55% у старых
- Среднее кол-во ходов стабильно 100-150
- White wins ≈ Black wins (≈ 50%)
- Способ побед меняется (не всё по очкам)
